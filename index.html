Featured project
Perception-to-Action Robotics Pipeline
A complete workflow: sense the scene → localize target(s) → execute actions → validate success.
Problem
Build a reliable pipeline that goes from perception to execution without brittle one-off scripts. The hard part is noise, timing, and verifying success.
Solution
Implemented a modular perception → decision → action → validation structure with clear interfaces for repeatability and fast debugging.
How
  • Separated perception, state estimation, and control into clean modules.
  • Step-by-step sequencing with success checks and safe fallback behavior.
  • Consistency in coordinate handling + deterministic parameters.
  • Visual validation and quick iteration using short clips/GIFs.
Robotics software Integration Testing
Proof + Status
  • Working GIF clips showing stable pipeline behavior across iterations.
  • Second-part videos will be added later.
  • Third extension was not completed.
Lessons learned
Robustness comes from verification: if you can’t define “success”, you can’t make it reliable. Timing, retries, and coordinate conventions matter as much as the main algorithm.
