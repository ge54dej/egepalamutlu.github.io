Bachelor Thesis

Towards Automated Chemistry Research: Making Laboratory Devices Intelligent and Automation-Compatible — proof-of-concept lab automation using a standardized device communication protocol (SiLA 2).

Demo video

Robot loads/unloads a microplate reader and runs an automated measurement workflow through SiLA 2 communication.

Key points (Problem → Solution → How → Proof → Lessons)
Problem
Lab devices often use different communication protocols, making end-to-end automation hard to integrate and maintain.
Solution
Built a SiLA 2 server–client setup to connect a Franka Emika Panda with a Thermo Fisher Varioskan LUX microplate reader and execute a full load → run → unload cycle.
How
  • Defined automation flow + requirements (manual vs automated steps, time/temperature-critical phases).
  • Implemented a custom SiLA 2 robot server with core commands: MoveToPose, GetPosition, Grip, Release.
  • Used the instrument’s SiLA interface via SkanIt server (connect, tray in/out, execute session, status checks, reports/errors).
  • Added synchronization for tray open/close using a safe delay to avoid collisions during loading.
Proof
Successful end-to-end test experiment: the system ran smoothly, with no unknown SiLA-related failures observed during the demo run.
Lessons learned
Reliable lab automation is mostly interfaces + timing + safety: robust sequencing, clear status checks, and mechanical constraints (gripper/fixtures) matter as much as software.
